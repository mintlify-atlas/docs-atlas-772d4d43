The TelemetryDeck Swift SDK lets you send analytics signals from your Apple platform apps to [TelemetryDeck](https://telemetrydeck.com). It's designed to be privacy-first: user identifiers are hashed on-device before transmission, and the SDK never collects personally identifiable information.

## How it works

TelemetryDeck uses a **signal** model: whenever something notable happens in your app, you send a named signal — optionally with key/value parameters — to the TelemetryDeck ingestion service. Your dashboard then lets you build charts and insights from those signals.

```swift
// Initialize once at app startup
let config = TelemetryDeck.Config(appID: "YOUR-APP-ID")
TelemetryDeck.initialize(config: config)

// Send a signal anywhere in your app
TelemetryDeck.signal("App.launchedRegularly")
```

The SDK batches signals locally and sends them in the background. You never need to manage network requests or caches manually.

## Platform support

The SDK supports all major Apple platforms:

| Platform | Minimum version |
|----------|----------------|
| iOS      | 12.0           |
| macOS    | 10.13          |
| watchOS  | 5.0            |
| tvOS     | 13.0           |
| visionOS | 1.0            |

<Note>
  The SDK requires Swift Package Manager (Xcode 15+ recommended). It is also available via CocoaPods using the `TelemetryDeck` pod.
</Note>
