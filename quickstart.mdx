---
title: "Quickstart"
description: "Install the TelemetryDeck Swift SDK and send your first analytics signal in a few minutes."
---

<Steps>
  <Step title="Install via Swift Package Manager">
    The recommended installation method is Swift Package Manager, which is built into Xcode.

    1. In Xcode, go to **File > Add Packages...**
    2. Enter the repository URL in the search field:
       ```
       https://github.com/TelemetryDeck/SwiftSDK
       ```
    3. Set the **Dependency Rule** to **Up to Next Major Version**
    4. Click **Add Package**
    5. When prompted, add the **TelemetryDeck** library to your app target

    **CocoaPods**

    If you use CocoaPods, add the following to your `Podfile`:

    ```ruby
    pod 'TelemetryDeck'
    ```

    Then run `pod install`.
  </Step>

  <Step title="Get your App ID">
    Sign up for a free account at [telemetrydeck.com](https://telemetrydeck.com). Once logged in, open the TelemetryDeck dashboard and navigate to **Set Up App** to find your App ID.

    You will need this identifier in the next step.
  </Step>

  <Step title="Initialize the SDK">
    Initialize TelemetryDeck once at app startup. For a SwiftUI `@main` app, do this in the `init()` method:

    ```swift
    import SwiftUI
    import TelemetryDeck

    @main
    struct MyApp: App {
        var body: some Scene {
            WindowGroup {
                ContentView()
            }
        }

        init() {
            let config = TelemetryDeck.Config(appID: "YOUR-APP-ID")
            TelemetryDeck.initialize(config: config)
        }
    }
    ```

    <Note>
      Initialize in `init()` of your `@main` App, not in `WindowGroup.onAppear` — the latter fires after window initialization and may cause missed signals.
    </Note>
  </Step>

  <Step title="Send your first signal">
    With the SDK initialized, send a signal from anywhere in your app:

    ```swift
    TelemetryDeck.signal("App.launchedRegularly")
    ```

    Signal names are arbitrary strings. Use a consistent naming convention (such as `Feature.action`) to keep your dashboard organized.
  </Step>

  <Step title="Verify in test mode">
    Debug builds automatically mark all signals as **testing signals**. To see them in the TelemetryDeck dashboard, activate **Test Mode** using the toggle in the dashboard toolbar.

    Testing signals are hidden in live mode, so they will not affect your production analytics data.
  </Step>
</Steps>

<Tip>
  The SDK behavior — default user, logging, session tracking, opt-out, and more — can be tailored to your needs. See the [Configuration](/configuration) page for all available options.
</Tip>
